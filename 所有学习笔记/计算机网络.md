#### 1、浏览器从输入网址到页面展示的过程

1. 输入 url ：浏览器会对 url 进行检测、补全，若未能形成完整的 url ，浏览器会将其作为搜索内容用默认搜索引擎进行搜索
2. DNS 域名解析：本机缓存、本地 DNS 服务器、DNS 根域名服务器、一级域名服务器、二级域名服务器；本机到本地 DNS 服务器是递归查询（一进一出），本地 DNS 服务器的查询是迭代查询（多进多出）
3. 建立 TCP 连接
4. 浏览器发送HTTP/HTTPS（后者需要建立 SSL/TLS 连接）
5. 服务器响应请求
6. 浏览器解析渲染页面
   - 处理HTML标记并构建DOM树
   - 处理CSS标记并构建CSSOM树
   - 将DOM和CSSOM合并为一个渲染树
   - 根据渲染树布局，以计算每个节点的几何信息
   - 将各个节点绘制到屏幕
7. TCP断开连接

#### 2、TCP 建立连接为什么三次握手

原因：**使得双方发送的数据包都能被确认（ACK，全称Acknowledge character， 确认字符）**

建立连接过程：

- 客户端：SYN	seq = x
- 服务端：SYN    seq = y    ACK = x+1
- 客户端：ACK = y+1

 解释：

- 如果没有第三次连接：只有客户端发送的数据包(x)得到了确认，而服务端发送的(y)没有得到确认；如果有了第三次连接：客户端和服务端发送的数据包均得到了确认；两次握手的话, 客户端有可能因为网络阻塞等原因会发送多个请求报文，延时到达的请求又会与服务器建立连接，浪费掉许多服务器的资源
- 为什么需要双方发送的数据包都能被确认：为了保证TCP的可靠性，如果其中一方没有得到确认，就会考虑是不是丢包，进而重发，实现可靠性
- 通过三次握手，就能确定双方的接收和发送能力是正常的，之后就可以正常通信了
